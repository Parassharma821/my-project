LIVE CHARACTER COUNTER – TEST CASES & VALIDATIONS

⸻

1. TEXTAREA: Input Field

Requirement:
	•	Must provide a textarea for user input.
	•	Placeholder text should guide the user.
	•	Should allow typing of spaces, letters, numbers, and newlines.

VALID TEST CASES:
	•	Textarea is visible on all screen sizes.
	•	Placeholder “Start typing…” appears initially.
	•	Text can be entered, edited, and deleted without issues.

INVALID TEST CASES:
	•	Textarea missing or hidden.
	•	Placeholder text not visible.
	•	Text cannot be typed or disappears on typing.

⸻

2. CHARACTER COUNT DISPLAY

Requirement:
	•	Must display live count of characters typed, including spaces and line breaks.
	•	Count should update instantly as the user types or deletes text.

VALID TEST CASES:
	•	Character count increments with each keystroke.
	•	Count decrements when text is deleted.
	•	Works with multiple lines and spaces.

INVALID TEST CASES:
	•	Count does not update.
	•	Count stuck at 0 or incorrect.
	•	Newlines or spaces ignored incorrectly.

⸻

3. INITIALIZATION

Requirement:
	•	Counter should initialize correctly on page load.
	•	Pre-filled text should show correct count immediately.

VALID TEST CASES:
	•	On page load, counter shows 0 if textarea empty.
	•	If textarea has default text, counter shows correct character number.

INVALID TEST CASES:
	•	Counter remains empty or shows incorrect number on page load.

⸻

4. RESPONSIVENESS

Requirement:
	•	Textarea and counter display should adapt to different screen sizes.
	•	Should remain readable and functional on desktop and mobile.

VALID TEST CASES:
	•	Layout adjusts on small screens without overflow.
	•	Text and count remain visible and accessible.

INVALID TEST CASES:
	•	Textarea or counter overlaps or becomes unreadable on smaller screens.
